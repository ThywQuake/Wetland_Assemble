## 四、 自定义分析工作流 (代码范例)

您可以参考以下逻辑在 Notebook 或新脚本中构建自己的分析：

```python
import pandas as pd
from wetland_analysis.utils.alignment import SpatioTemporalAligner
from wetland_analysis.utils.geospatial import create_30m_grid

# 1. 定义研究区 (ROI) 和参考网格 (例如 30m)
# 范围格式: [西, 南, 东, 北]
roi_bounds = (-65.0, -3.0, -64.0, -2.0) 
ref_grid = create_30m_grid(roi_bounds)

# 2. 定义目标时间轴 (例如 2018 年全年)
target_time = pd.date_range("2018-01-01", "2018-12-01", freq="MS")

# 3. 初始化对齐器
aligner = SpatioTemporalAligner(ref_grid, target_time_index=target_time)

# 4. 添加并自动对齐数据集
# 系统会自动处理 GWD30 的 15m 偏移和 SWAMPS 的日/月聚合
aligner.add_dataset("gwd30", year=2018)
aligner.add_dataset("glwd_v2")

# 5. 执行时间轴对齐 (将静态数据重复到每个月)
aligner.align_temporally("glwd_v2", method="repeat")

# 6. 获取合并后的 Xarray Dataset 进行后续分析
ds_combined = aligner.combine()
print(ds_combined)
```

## 五、 常见问题 (Troubleshooting)

*   **维度命名错误**：加载器已内置 `COORD_MAP`，会自动将 `latitude/y` 统一为 `lat`。如果遇到新的命名，请在 `src/wetland_analysis/data/loader.py` 的 `COORD_MAP` 中添加。
*   **内存溢出**：代码已默认开启 Dask 懒加载（Chunks）。如果处理全球尺度数据，请确保使用 `ds.compute()` 之前进行了空间裁剪 (ROI)。
*   **零重叠报错**：如果提示 `ZERO OVERLAP`，请检查您的 `roi_bounds` 是否超出了该数据集的 `Spatial Range`（见对应数据集的 `.md` 文件）。
