# 湿地项目文档样式指南

## 1. 文档结构

### 1.1 文件命名规范

使用以下格式：
```
YYYY-MM-DD-[type]-[description].md
```

类型前缀：
- `plan-`: 实施计划
- `feat-`: 功能规范
- `brainstorm-`: 头脑风暴
- `review-`: 审查报告
- `guide-`: 使用指南
- `spec-`: 技术规范

示例：
- `2026-03-04-plan-wetland-dataset-analysis-details.md`
- `2026-03-04-feat-wetland-dataset-evaluation-plan.md`

### 1.2 标题层级

使用以下层级结构：

```markdown
# 一级标题 - 文档主标题

## 二级标题 - 主要章节（使用中文数字：一、二、三）

### 三级标题 - 子章节（使用阿拉伯数字：1.1、1.2）

#### 四级标题 - 小节（使用阿拉伯数字）

##### 五级标题 - 细节说明
```

### 1.3 章节编号

- 一级章节：使用中文数字（一、二、三）
- 二级章节：使用阿拉伯数字（1.1、1.2）
- 三级章节：使用阿拉伯数字（1.1.1、1.1.2）
- 表格编号：表X.X（图X.X）
- 代码块：使用语言标识

## 2. 表格规范

### 2.1 表格结构

所有表格必须包含：
1. 表格标题（位于表格上方）
2. 表格编号（格式：表X.X）
3. 表头行（加粗）
4. 对齐一致的列

### 2.2 表格示例

**表1.1 数据集基本信息**

| 数据集 | 分辨率 | 时间范围 | 格式 |
|--------|--------|----------|------|
| GWD30 | 30m | 2013-2022 | GeoTIFF |
| GIEMS-MC | 0.25° (约27km) | 1993-2007 | NetCDF |

### 2.3 表格设计原则

- 列数：建议3-7列
- 列宽：保持一致
- 对齐：数字右对齐，文字左对齐
- 间距：使用单个空格分隔

## 3. 命名约定

### 3.1 数据集名称

使用统一的缩写形式：

| 缩写 | 全称 | 说明 |
|------|------|------|
| GWD30 | Global Wetland Dataset 30m | 全球30米湿地数据集 |
| GIEMS-MC | GIEMS-Monthly Climate | 月度气候洪水淹没数据集 |
| GLWD | Global Lakes and Wetlands Database | 全球湖泊和湿地数据库 |
| WAD2M | Wetland Area and Dynamics for Methane | 甲烷建模湿地面积和动态 |
| SWAMPS | Surface Water Microwave Product Series | 微波地表水产品系列 |
| G2017 | Global 2017 Wetland | 2017年全球湿地 |

### 3.2 分辨率表示

统一使用以下格式：
- `30m`：原生高分辨率
- `0.25° (约27km)`：粗分辨率网格
- `0.05° (约5km)`：中分辨率网格
- `30s (约1km)`：高分辨率网格

### 3.3 文件和变量命名

- 文件名：使用 kebab-case（短横线分隔）
- 变量名：使用 snake_case（下划线分隔）
- 常量：使用 UPPER_SNAKE_CASE

## 4. 颜色规范

### 4.1 颜色代码格式

所有颜色使用6位十六进制格式：
- ✅ `#2E7D32`（正确）
- ❌ `#F5F5F5`（应为6位）
- ❌ `#2E7D`（不完整）

### 4.2 语义颜色映射

| 语义 | 颜色 | 代码 |
|------|------|------|
| 湿地/水体 | 蓝色系 | `#1565C0` |
| 森林/植被 | 绿色系 | `#2E7D32` |
| 干旱/裸地 | 棕色系 | `#8D6E63` |
| 城市/建筑 | 灰色系 | `#616161` |
| 趋势增加 | 红色系 | `#C62828` |
| 趋势减少 | 蓝色系 | `#1565C0` |
| 无显著趋势 | 灰色系 | `#9E9E9E` |

## 5. 代码块规范

### 5.1 语言标识

所有代码块必须指定语言：

```python
import xarray as xr

data = xr.open_dataset("wetland.nc")
```

```yaml
datasets:
  gwd30:
    path: /data/gwd30/
```

```bash
uv sync
pytest tests/
```

### 5.2 代码块标题

复杂代码块可以添加标题：

```python
# 图1.1 数据加载示例
import xarray as xr
```

## 6. 图表规范

### 6.1 地图

- 使用Cartopy投影
- 添加比例尺和北向箭头
- 使用语义颜色（见4.2节）

### 6.2 图表

- 图表标题：图X.X 标题
- 图例位置：右上方或下方
- 坐标轴标签：包含单位和维度
